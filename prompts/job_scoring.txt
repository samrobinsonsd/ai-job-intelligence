=========================================================
EVALUATION RULES
================

Do not invent missing information.

Do not assume responsibilities from title alone.

Use the full 0-100 range.

Most roles should not be High Value.

A role should score 80+ only when several strong signals are explicit in the job data.

A role should not score 75+ unless at least three of these are clearly true:

* Senior-level or mid-senior role
* Strong technical alignment with UCaaS, CCaaS, telecom, AI, APIs, integrations, or cloud
* Clear customer-facing presales, solution consulting, demo, discovery, POC, or solution architecture work
* Remote or hybrid
* Competitive compensation is explicitly listed
* Strong company or industry alignment

If compensation is unknown, the maximum score is 80.

If the role is customer-facing but lacks technical depth, the maximum score is 70.

If the title sounds relevant but the description does not show presales, demos, discovery, POCs, solution design, or customer technical ownership, the maximum score is 74.

If the role is explicitly junior or entry-level, the maximum score is 70 unless there is an exceptional reason.

If the role is mostly internal IT, support, help desk, NOC, desktop support, field technician, or MSP support, the maximum score is 55.

If the role is primarily software engineering with little customer-facing work, the maximum score is 65.

Use 70 as the default for plausible but incomplete roles.

COMPENSATION EXTRACTION RULES
=============================

Extract compensation from the job description.

Return compensation_min and compensation_max as integers.

Return 0 for both values if no explicit numeric compensation is stated.

Return compensation_type as exactly one of:

* "Base Salary"
* "OTE"
* "Salary"
* "Hourly"
* "Unknown"

Do not return lowercase variants.

Use "OTE" only if the posting explicitly says OTE, on-target earnings, target earnings, or similar sales-compensation language.

Use "Base Salary" if the posting explicitly says base salary or base pay.

Use "Salary" if a numeric salary range is stated but the posting does not clearly identify it as base salary or OTE.

Use "Hourly" only for hourly compensation.

If both base salary and OTE are provided, prefer OTE for compensation_min, compensation_max, and compensation_type.

Ignore bonuses, equity, 401k contributions, benefits, signing bonuses, commission percentages, and unrelated dollar amounts.

If compensation is vague, such as “competitive salary” without numbers, return:
compensation_min: 0
compensation_max: 0
compensation_type: "Unknown"

SCORING GUIDE
=============

90-100:
Exceptional fit. Clear seniority, strong technical alignment, strong customer-facing presales responsibilities, strong industry fit, remote or hybrid flexibility, and competitive compensation.

Weak fit. Generic, too junior, mostly support/internal, limited customer-facing ownership, or poor technical alignment.

Poor fit. Clearly outside the target career path.
